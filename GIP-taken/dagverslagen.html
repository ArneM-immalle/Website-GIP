AURELIUM

Aurelium is een expert in het bedenken van ICT-oplossingen voor KMO’s voor zo goed als elk aspect van hun ICT-infrastructuur. Wanneer een KMO een probleem heeft, belt deze naar de helpdesk. Zij zullen een ticket maken met daarin het probleem geschetst. De System Engineers kunnen deze tickets bekijken via het programma "Microsoft Dynamics NAV Classic" en deze proberen op te lossen. De taak van de Software Engineers zal ik deze week meevolgen.

2019-09-30

De eerste dag heb kennis gemaakt met veel nieuwe software zoals:

  • Remote Desktop Connection Manager (voor het op afstand beheren van servers van klanten);
  • TeamViewer (beheer van clientcomputers van klanten);
  • Veeam (back-up oplossing);
  • Carbonite (back-up oplossing).

De tickets die Bert (mijn stagebegeleider) en ik vandaag hebben bekeken waren onder andere:

  • een mailadres dat spam verstuurt → geblokkeerd via Microsoft Partner Center
  • printserver die geen printers meer vindt door Windows update → update gedeïnstalleerd
  • bedrijf waarbij firewall was aangepast → firewall terug opengezet
  • kassa kon geen verbinding maken met printer → Virtual Port Driver geïnstalleerd
  • gebruiker moest worden toegevoegd aan Acitive Directory en Office 365 £3 en Advanced Threat Protection licentie geactiveerd bij deze gebruiker via Microsoft 365 Admin Center
  • back-up services failed → Manueel checkpoint (Hyper-V) maken lukt niet → herstart
  • printer moet automatisch landscape printen i.p.v. portrait → test gedaan bij andere printers → overal doet hij automatisch, behalve bij 1 niet
  • User Certificate aanvragen → inloggen met domainadmin anders lukte niet
  • dagelijkse back-up failed → vssadmin list writers in Powershell uitgevoerd → sqlserverwriter gaf een error → server herstart → werkte
  • ...

Ik vond het een boeiende eerste stagedag. Mijn stagebegeleider, Bert, heeft mij goed verwelkomd. Hij vertelde mij over het bedrijf en hoe zij werken. Tijdens het oplossen van tickets heeft hij me telkens de nodige achtergrondinfo gegeven o.a. over de software.

2019-10-01

Ook vandaag heb ik nieuwe programma's leren kennnen:

  • TrendMicro (een antivirus- en online beveiligingsprogramma);
  • Citrix (IT-afdelingen respectievelijk gecentraliseerde desktops en applicaties hosten → gebruikers overal toegang tot applicaties);
  • Mozy (back-ups centraal beheren);
  • Event Viewer (gebeurtenislogboeken);
  • Group Policy (beheert werkomgeving van gebruikersaccounts en computeraccounts).

De tickets die Bert en ik vandaag hebben bekeken waren onder andere:

  • gebruiker verwijderd
    • TrendMicro: licentie gebonden aan computer → Via TrendMicro.com bekeken welke computer van deze gebruiker was → op deze computer software verwijderd
    • Office: licentie gebonden aan gebruiker → gebruikersaccount disabled (en niet verwijderd) → mailbox nog 30 dagen beschikbaar
  • bedrijf kon niet aanmelden met Citrix → oorzaak nog niet gevonden
  • bedrijf met eigen mailserver → na inloggen → pagina niet gevonden → blijkbaar waren die nacht updates geïnstalleerd → computer zou volgens Event Viewer 15 minuten na installatie herstarten, maar er was nog een gebruiker aangemeld → updates konden niet gebeuren → met Group Policy ingesteld om te voorkomen voor volgende keer
  • van Mozy overschakelen naar Carbonite → oude backups zolang mogelijk bewaren → wekelijks controleren
  • Veaam: daily backup → werkte eerst niet, plots wel
  • printer (van een ticket van gisteren) printte plots wel automatisch landscape
  • ...

Vandaag heb ik met de hulp van Bert verbinding kunnen maken met een server binnen Aurelium via de Remote Desktop Connection Manager. Hierop mocht ik een beetje "spelen". Ik heb weer kennis gemaakt met nieuwe programma's. Group Policy vond ik zeer interessante software.

2019-10-02

Vandaag heb ik vooral gewerkt aan mijn Case-study. Ik heb de opdracht gekregen van Bert om via een PowerShell script de best practices te scannen van alle servers binnen een netwerk en te tonen. Best practices zijn de richtlijnen die worden beschouwd als de ideale manier om onder normale omstandigheden een server te configureren. Het is best interessant om op deze manier te ontdekken wat er allemaal mogelijk mee is. Dit script zou binnenkort gebruikt worden door Aurelium. Ik kan mijn script testen op één van hun servers waar ik gisteren toegang op heb gekregen.

2019-10-03

Vandaag hebben Bert en ik zoals gewoonlijk tickets behandeld. Eén van de interessantste tickets ging over een probleem binnen een Storage Area Network (SAN), er was namelijk een schijf binnen de opslag gecrasht. Deze zou dus vervangen moeten worden door een medewerk van Dell. Om Dell te contacteren had Bert een rapport (DGO-file) nodig dat gegenereerd werd via Dell Equallogic Group. Deze heeft hij vervolgens doorgestuurd naar Dell, zij komen de harde schijf morgen normaal vervangen.

Ik vond dit een interessant ticket omdat we vorig schooljaar al veel geleerd hebben over RAID. In dit geval ging het over RAID 50, dat is een combinatie van RAID 5 en mirroring.

Ook vandaag heb ik veel gewerkt aan mijn script. Ik ben de dag geëindigd met een test waarbij de rollen van alle servers binnen het netwerk gescand werden. Vervolgens zou in een HTML-file het resultaat van de inbreuken op de best practices verschijnen. Morgen zullen we zien of het gewerkt heeft...

2019-10-04

Vandaag heb ik in het grote script waar Aurelium al 5 jaar lang aan werkt mogen kijken. Het bevatte meer als 5000 regels en is enkel bedoelt om server-instellingen te controleren. Er wordt niets aangepast aan de server-instellingen, dat vinden ze te gevaarlijk. Binnenkort zal hier mijn BPA-script (mijn case-study) in komen te staan.

De tickets die Bert en ik vandaag hebben bekeken waren onder andere:

  • Printserver werkte niet meer → deze update ge-deinstalleerd en geblokkeerd
  • Gebruiker kon niet inloggen bij (Microsoft Azure) ObD Quality Kick → een permissie "User.Read.All" toegevoegd + Admin grant gedaan
  • Virtuele server met minder dan 100 GB vrij → gekeken of er voldoende ruimte beschikbaar was op de fysieke server (ja) → via Hyper-V harde schijf meer ruimte gegeven + partition vergroot bij disk management
